Setup Gmail

1 Create Google Cloud Project

You need to create a Google Cloud project to access Gmail API:

  • Go to Google Cloud Console
  • Click "New Project" and give it a name (e.g., "Email Analyzer")
  • Wait for the project to be created
2 Enable Gmail API

Enable the Gmail API for your project:

  • In the Google Cloud Console, go to "APIs & Services" → "Library"
  • Search for "Gmail API"
  • Click on it and press "Enable"
3 Create OAuth 2.0 Credentials

Create credentials to access the API:

  • Go to "APIs & Services" → "Credentials"
  • Click "Create Credentials" → "OAuth 2.0 Client ID"
  • If prompted, configure the OAuth consent screen first:
    • Choose "External" user type
    • Fill in app name: "Email Space Analyzer"
    • Add your email as a test user
  • For Application type, choose "Desktop application"
  • Name it "Email Analyzer Desktop"
  • Click "Create"
4 Download Credentials File

Save the credentials file:

  • Click the "Download" button (⬇️) next to your newly created credentials
  • A JSON file will be downloaded
  • Rename this file to gmail_credentials.json
  • Save it in the same folder as this application
🔑 Keep This File Safe
This file contains your API credentials. Don't share it publicly or commit it to version control.
5 Upload Credentials

Upload your credentials file here:

Select the gmail_credentials.json file you downloaded

Setup Microsoft Outlook

1 Account Requirements

This application works with the following Microsoft account types:

  • Personal Microsoft accounts: @outlook.com, @hotmail.com, @live.com
  • Microsoft 365 accounts: Work or school accounts with admin permission
⚠️ Note for Work/School Accounts
If you're using a work or school account, your organization's administrator may need to approve this application first.
2 Authentication Process

When you click "Start Authentication", you'll see:

  • A device code and verification URL
  • Instructions to open the URL in your browser
  • A prompt to enter the device code
  • Microsoft's login page to enter your credentials

This is the official Microsoft OAuth2 flow and is completely secure.

3 Permissions Required

The application will request permission to:

  • Read your profile: To identify your account
  • Read your mail: To analyze email metadata (sender, date, size, subject)
🔒 Privacy Notice
We only read email metadata. Email content is never accessed or stored.
